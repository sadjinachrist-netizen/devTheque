Maîtriser les Server Components en React 19

WEB DEV
Lucas Rivière Senior Frontend Engineer
calendar_today 12 Oct 2024

Introduction à React 19

React 19 marque une étape historique dans l'évolution de la bibliothèque avec l'introduction stable des Server Components. Cette technologie change radicalement notre manière de concevoir l'hydratation et le transfert de données entre le serveur et le client.

Contrairement aux composants traditionnels, les Server Components ne s'exécutent que sur le serveur. Cela permet de réduire drastiquement le poids du bundle JavaScript envoyé au navigateur, tout en améliorant les performances de chargement initial.

"Les Server Components ne sont pas seulement une amélioration de performance, c'est un nouveau paradigme qui fusionne le meilleur du rendu serveur avec l'interactivité du client."

Implémentation d'un composant serveur

Voici comment structurer un composant qui récupère des données directement depuis une base de données sans passer par une API intermédiaire : 

// app/posts/page.js
import { db } from '@/lib/database';

async function BlogPost({ id }) {
  const post = await db.posts.findUnique({ where: { id } });

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}

L'utilisation de async/await directement dans la fonction du composant simplifie énormément la gestion du state asynchrone, éliminant le besoin de useEffect complexes pour le fetching initial.
